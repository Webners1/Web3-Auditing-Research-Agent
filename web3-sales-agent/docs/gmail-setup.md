# Gmail MCP Setup Guide
**Time required:** ~10 minutes  
**Account:** muzammilsiddiqui001@gmail.com

After completing this setup, the agent will automatically create Gmail drafts after generating each pitch. You review and click Send — no copy-paste needed. Replies are also monitored automatically.

---

## What You'll Have When Done

- Agent creates a Gmail draft after every pitch
- Draft appears in your Gmail Drafts with correct subject, body, and To address
- You open Gmail → Drafts → review → click Send
- Agent can check your inbox for replies and update the pipeline status

---

## Step 1 — Create a Google Cloud Project

1. Go to [https://console.cloud.google.com](https://console.cloud.google.com)
2. Click **Select a project** → **New Project**
3. Name it: `web3-rabbit-outreach`
4. Click **Create**

---

## Step 2 — Enable the Gmail API

1. In Google Cloud Console, go to **APIs & Services → Library**
2. Search for **Gmail API**
3. Click **Enable**

---

## Step 3 — Configure the OAuth Consent Screen

1. Go to **APIs & Services → OAuth consent screen**
2. Select **External** (even though this is your personal account)
3. Fill in:
   - App name: `Web3 Rabbit`
   - User support email: `muzammilsiddiqui001@gmail.com`
   - Developer contact email: `muzammilsiddiqui001@gmail.com`
4. Click **Save and Continue**
5. On the **Scopes** page, click **Add or Remove Scopes**
6. Add these two scopes:
   - `https://www.googleapis.com/auth/gmail.compose`
   - `https://www.googleapis.com/auth/gmail.readonly`
7. Click **Save and Continue** through the rest
8. On the **Test Users** page, add `muzammilsiddiqui001@gmail.com`
9. Click **Save and Continue** → **Back to Dashboard**

---

## Step 4 — Create OAuth Credentials

1. Go to **APIs & Services → Credentials**
2. Click **+ Create Credentials → OAuth client ID**
3. Application type: **Desktop app**
4. Name: `web3-rabbit-local`
5. Click **Create**
6. Click **Download JSON** on the popup
7. The file will be named something like `client_secret_XXXXXXXXXX.json`

---

## Step 5 — Place the Credentials File

1. Create the directory:
   ```powershell
   New-Item -ItemType Directory -Force "C:\Users\tahan\.gmail-mcp"
   ```

2. Rename the downloaded file to `gcp-oauth.keys.json` and move it:
   ```powershell
   Move-Item "C:\Users\tahan\Downloads\client_secret_*.json" "C:\Users\tahan\.gmail-mcp\gcp-oauth.keys.json"
   ```

3. Verify the file is in place:
   ```powershell
   ls C:\Users\tahan\.gmail-mcp\
   # Should show: gcp-oauth.keys.json
   ```

---

## Step 6 — Authenticate (One-Time)

Run this command in a terminal (not inside Claude):

```powershell
npx @gongrzhe/server-gmail-autoauth-mcp auth
```

This will:
1. Open a browser window
2. Ask you to log in with `muzammilsiddiqui001@gmail.com`
3. Show a Google OAuth consent screen — click **Allow**
4. Generate `credentials.json` in `C:\Users\tahan\.gmail-mcp\`

After this, the agent has persistent access. You won't need to re-authenticate unless you revoke access.

---

## Step 7 — Restart Claude Code

Close and reopen the Claude Code VS Code extension. The Gmail MCP will be loaded automatically from `.mcp.json`.

To verify it's working, ask the agent:
```
list my Gmail labels
```

You should see your Gmail labels returned.

---

## What the Agent Can Do With Gmail

| Tool | What It Does | When Used |
|------|-------------|-----------|
| `draft_email` | Creates a draft in your Gmail Drafts folder | After each pitch is generated |
| `search_emails` | Searches inbox with Gmail syntax | Checking for replies |
| `read_email` | Reads a specific email thread | Reading protocol team responses |
| `list_email_labels` | Lists your Gmail labels | Setting up label organization |

The agent does **NOT** use `send_email` — you always click Send yourself.

---

## Troubleshooting

**"Cannot find module" error:**
```powershell
npm install -g @gongrzhe/server-gmail-autoauth-mcp
```

**"Authentication failed" or token expired:**
```powershell
npx @gongrzhe/server-gmail-autoauth-mcp auth
```
Re-run the auth command and log in again.

**MCP not appearing in Claude Code:**
Check that `.mcp.json` is in `d:/web3-auditing-agent/` and restart VS Code.
